# Small demo builds — 3 OS × 3 colour depths (9 images)

Generated by `builds/gen_configs.py` (edit that + re-run to regenerate the
`gen-*.json` configs), then built with:

```sh
cd /home/dani/repos/MacAtrium
for c in builds/gen-*.json; do tools/atrium-tool/target/release/atrium image --config "$c"; done
```

Every image is a small, bootable MacAtrium disk with the **same handful of five
classic games** and a curated catalog. **Prince of Persia is always included.**

## The three colour "sets"

Following the conventions in `data/templates.json`:

| Set | `art_depths` | Launcher partition | Art bound |
|-----|--------------|--------------------|-----------|
| **1-bit** (B&W)      | `1`      | 512 / 384 KB   | default |
| **256-colour**       | `1,8`    | 1024 / 768 KB  | 384×384 px (memory-safe colour) |
| **Full colour** (millions) | `1,8,24` | 3584 / 3072 KB | default (24-bit ≈ 1.4 MB PICTs) |

The launcher picks the art variant matching the screen depth at runtime; lower
depths are always kept as fallbacks.

## Notes / caveats

- **6.0.8** deploys the launcher **as the Finder** (System 6 appliance); 7.1 and
  7.5.5 launch it from **Startup Items**.
- **24-bit "full colour"** art is *computed, not emulator-verified* (Snow tops
  out at 8-bit). On an 8-bit screen these builds degrade gracefully to the 8-bit
  variant. The full-colour builds use a 3 MB launcher partition — on the System 6
  / Mac LC target that needs a machine with enough RAM.
- Not yet booted in an emulator; verify with the Snow / q800 harness if desired.
